Build context

Architecture

Six custom Odoo modules: studio_core (instructors, rooms, class types), studio_booking (booking, waitlist, attendance), studio_membership (packs, contracts, family), studio_payment (provider abstraction), studio_api (JWT REST), studio_website (public site + portal).

Migration

Mandatory migration from PerfectGym: active members, active contracts, forward bookings, pack credit balances. Historical attendance & transactions can be deferred. Cutover window: 1 weekend with read-only fallback.

Phasing

Phase 1 — backend foundation (5–6 wk). Phase 2 — public site & member portal (3–4 wk). Phase 3 — mobile app (8–10 wk, parallel). Phase 4 — migration & cutover (2 wk). Total: ~18–22 weeks with two developers.
